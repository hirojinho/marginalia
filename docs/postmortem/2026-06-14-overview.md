# Overnight pipeline postmortem — 2026-06-14

## What happened

The overnight fleet ran 19 real tickets against the claw-study agent pipeline. Nine shipped cleanly (including 6 with trivial diffs and deeper feature work like courses CRUD, retrieval-practice banding, and r9-probe-infra), one failed on a budget overrun (r4-confidence-log, diff 355>350), and six more aborted at the gate with exit 10 — all triggered by a systemic `! grep -q` verifier inversion that made the pre-baseline checker falsely report "feature already present." One ticket, sm2-spaced-review, had already been manually salvaged and deployed on 2026-06-03 after a build-break failure but was left stranded in `specs/failed/` (bookkeeping never completed). The single genuine, fresh, unshipped hard failure was **r9-full-pipeline** — a 495-line implementation that never made it past the gate.

## What went wrong — r9

The gate killed r9 at **exit 12** (`go test`). Four defects span two layers:

**EXECUTOR (3 defects):**
1. **Bare `%q` syntax** — `claw-cli/main.go:1716`: `fmt.Sprintf(%q, learnerAnswer)` missing string quotes around the format verb. Compilation trivially fatal. Exit: the build step short-circuited *all* `claw-cli` tests, hiding two more failures underneath.
2. **NULL `learner_answer` scan** — `TestProbeShowByID` crashed on a `NULL` column scanned into a bare `string`; fix was `sql.NullString`. Only surfaced after the syntax fix unblocked the full test suite.
3. **`session_id=0` FK violation** — `TestProbeRecord` inserted a record with `session_id: 0`, which violated the foreign-key constraint. Fixed by passing `nil` (defaults to active session).

**SPEC (1 defect):**
4. **Rule-6 rewrite vs ADR-0020** — The r9 spec *ordered* a verbatim rewrite of Rule 6 that dropped two ADR-0020 guard strings (`"ONE light move"`, `"the prediction IS your opener"`) and introduced the explicitly prohibited phrase `"does NOT license skipping"`. The executor obeyed the spec exactly. The pre-existing `TestRule6OneLightOpener` guard test caught all three violations. Root cause: **SPEC**, not executor — the spec cited ADR-0020 while contradicting it, and Task 4 never updated the guard test it was breaking.

## The drift

The spec↔ADR-0020 conflict is the [[claw-study-pipeline-drift-rootcause]] genre realised live. ADR-0020 mandates that when the retrieval queue is empty, the prediction *is* the opener — no forced recall round. The r9 spec rewrote Rule 6 to say: if a task was completed, "you MUST still open with 2–3 targeted short-answer questions … it does NOT license skipping the recall." These are contradictory pedagogies. The syntax fix was trivial; the Rule-6 conflict was not auto-fixable — it required a human decision. The decision was to **preserve ADR-0020** and weave the probe-tool instructions *around* the invariant guard phrases rather than replacing them.

## The fix and re-ship

The reconciled Rule 6 was applied on the r9 branch alongside the three executor fixes (`"%q"`, `sql.NullString`, `nil` session_id). At this point a new blocker surfaced: the merge-base had **moved**. Origin/main had advanced at 17:02 (commits `fd70cec` for rule9/11 pedagogy and `acd56a9` for plan-active) — after r9's 05:00 run branched from the now-stale `57b886f`. A direct FF-merge into main would have been rejected. Salvage: merge `origin/main` into the r9 branch; `agent/sandbox.go` and `claw-cli/main.go` auto-merged clean (r9 touched rule6, main touched rule9/11 — zero overlap); only `claw-cli/main_test.go` conflicted (both appended tests → kept both blocks). The **merged result** (`8e211c0`) was re-gated: gate **PASSED exit 0** (pre-baseline fail-as-expected, build, go test, post-acceptance). TestRule6OneLightOpener passed; both pedagogy edits intact. FF-merged into main as `1ee685d`, spec moved to `specs/done/`, deployed. Prod `/debug/health` returned **200**.

## Pipeline hardening

Fixes pushed in `852a9dd`:
- **Verifier non-inversion rule** — `AGENTS.md` (planning rules) now mandates the `if grep -q …; then echo UNEXPECTED; exit 0; fi` / `exit 1` form and forbids `! grep -q` (the exit-code inversion behind the 6-ticket exit-10 cluster). This is author-side guidance; orchestrator hard-fail enforcement remains an open recommendation (below).
- **Spec↔ADR reconciliation rule** — `AGENTS.md` now requires a spec that changes ADR/guard-test-protected behavior to cite the ADR, reconcile *meaning* (not just terminology), and update or retire the guard test in-spec. The harvested `pipeline-postmortem` skill enforces the analysis side: read the spec AND its cited ADRs before classifying a failure EXECUTOR vs SPEC.
- **Executor git-add hygiene** — `implement-from-spec` now bans `git add -A`/`git add .` and requires named-path adds. r9's original run swept 5 editor `.bak` files into the commit (`claw-cli/main.go.bak` through `.bak5`); they were removed and a `.gitignore` pattern (`*.bak`/`*.orig`) added.
- **sm2 bookkeeping** — moved the sm2 spec from `failed/` to `done/` to reflect the already-deployed salvage.

Open recommendations (not yet done):
- **Orchestrator**: hard-fail on any `! grep` verifier rather than warning-and-aborting (exit 10 is ambiguous — "already satisfied" looks the same as "verifier broken").
- **Failed runs orphan specs**: r9 had no spec dir at all after failure (neither `done/` nor `failed/`); the orchestrator must move the spec to `failed/` on any non-zero gate exit.
- **Live-dev racing the queue**: main advanced between r9's queue time and its salvage — any same-day live commit creates a merge-base hazard. The pipeline should **re-gate-on-merge** as a standard step, not a surprise discovery.

## What we learned about driving Pi

Five meta-log harvests from running the postmortem itself:
1. **Timeout belt is non-negotiable.** macOS has no `timeout(1)`; the pm-p1 triage sweep hung Pi's EventStream for 24 minutes in S-state (provider stall, zero sockets, zero billing). Built `raw/pirun.sh` with a hard timeout+retry wrapper. Every local Pi invocation now goes through it.
2. **Read-only forensics works first try.** The pm-smoke run proved `-xt write,edit` held; Pi ssh'd to the VPS, enumerated results, and returned "26" with no writes attempted. Mode flags are reliable.
3. **Must read the spec + ADRs before classifying.** Pi's r9 deep-dive was mechanically excellent but labelled the Rule-6 failure EXECUTOR — it had the diff and gate.log but *not* the spec that ordered the rewrite. The postmortem skill now requires loading the spec and every ADR it cites before layer-classification, with a mandatory "is this regression spec-intended?" check.
4. **Full test suite after any build fix.** The `%q` syntax error short-circuited all `claw-cli` tests in the gate.log; fixing the build unmasked two more failures (NULL scan, session FK). Never trust a truncated gate.log as the complete defect list.
5. **Session flag collisions are real.** `--session-id` and `--continue` conflict; the first pm-p4 apply call failed silently on invalid flags. Retry with a fresh `--session-id` reading the proposal file succeeded.
