## [Unreleased]

### Added
- **R4 — Confidence trajectory persistence (`confidence_log`).** Rule-3 confidence values are now persisted: new `confidence_log` table (`session_id`, `kc_id`, `value`, `source`, `created_at`, `raw_text`), a `log_confidence` agent tool, and a `claw-cli confidence trajectory|recent|schema` subcommand. Foundation for the retrieval queue and learning-curve views. Shipped 2026-05-27 (gate green; diff-cap overrun merged manually). Vocabulary is being standardised to `knowledge_component_id` next — see [ADR 0007](docs/adr/0007-knowledge-component-as-atomic-note.md).
- **Pedagogical rules baked into both code paths (8 rules).** Constructivist + cognitive-load scaffolds applied on every chat turn. Rules 1–5 (no lecturing / ask prior knowledge / confidence check / connect to prior knowledge / Bloom progression) plus three retrieval-side rules added 2026-05-19: Rule 6 (session-open free-recall, Roediger & Karpicke 2006), Rule 7 (pre-Read prediction, Slamecka 1978 generation effect), Rule 8 (max 3 new terms per turn, Sweller 1988 cognitive load). Reaches legacy `/chat` via `CLAUDE.local.md` + `agent.toolsAndRulesPrompt`; reaches Pi `/chat-v2` via the AGENTS.md template generated by `agent/sandbox.go` (pedagogy block sits last for maximum LLM weight).
- **AGENTS.md template overhaul.** Pi now sees, in order: course memory (user profile / observations / course context / feedback rules — loaded via `claw-cli memory load`) → session block → skill index → JSON-only plan tools → pedagogy rules + interest-log surfacing prompt.
- **Skill index in AGENTS.md.** One-line trigger hint per skill (`course-study-path`, `resource-orientation`, `study-step-complete`, `study-notes`, `pair-coding`, `by-hand`) so Pi can pick the right pedagogy without re-scanning every SKILL.md.
- **`claw-cli plan status --course <c>` subcommand.** Compact, agent-friendly view of current plan task status with `[x] #N <title>` lines. `#N` linear index is the same one used by `claw-cli plan toggle --task`, so Pi can read and mutate without translating between views.
- **JSON-only plan enforcement.** AGENTS.md template now explicitly instructs Pi that `data/plans/<course>.json` is the only source of truth and any `study-plan.md` files are retired. Per-turn `<plan_state>` block prepended to user messages for course-scoped sessions (handler/chat_v2.go `buildPiPrompt`) puts fresh authoritative state inside the conversation history where it can't be ignored on `--continue`.
- **SSE keepalive on `/chat-v2`.** `streamPiTurn` now emits `: keepalive\n\n` SSE comment frames every 15 s during idle gaps, preventing Cloudflare's tunnel from reaping long-running streams during slow LLM/tool waits.
- **Empty-response guard.** When Pi finishes a turn without emitting any token or reasoning content, the handler now emits an explicit `error` SSE event before `done`, surfacing the bail to the UI instead of closing silently.
- **Turn-timeout error event.** When the 10 min per-turn deadline fires (up from 5 min), the handler emits a friendly `error` SSE event before returning, so the browser sees "Turn timed out…" instead of "Error in input stream".
- **Pi lock diagnostics.** `AcquirePiLock` returns the age of an existing lock when acquisition fails; the 409 handler logs `pi_lock_conflict session_id=N existing_lock_age=…` so retry-vs-leak can be diagnosed from journal output.
- **`resource-orientation` skill: default trigger.** Pi now auto-runs orientation before any 🔴 Read task in the plan, not only when the user explicitly says "I'm starting X". Orientation output gained a mandatory closing "predict the key idea" prompt (generation effect).
- **`study-step-complete` skill: free-recall Step 0.** Before loading any context, Pi asks the user to recall in their own words the 3 key points from the just-finished task. Anchoring then uses the recall — not the page — as the conversational driver.
- **`course-study-path` skill: generative Reflect + Bloom tags (MANDATORY).** Reflect tasks must withhold the answer (generation effect); each Reflect task carries a Bloom tag (`[B-A]` Apply / `[B-N]` Analyze / `[B-E]` Evaluate / `[B-C]` Create); a cluster is not well-formed unless at least one task hits Evaluate or Create.

### Changed
- **Per-turn timeout 5 min → 10 min** (`handler.piTurnTimeout`) and matching `piLockTTL` 5 m 30 s → 10 m 30 s. Course-planning turns that involved PDF extraction were hitting 5 min cleanly; 10 min comfortably accommodates them.
- **`streamPiTurn` is ctx-aware.** Added a `case <-ctx.Done()` arm so the function returns immediately when the client disconnects or the deadline fires. Previously the handler could be blocked on writes to a dead connection, holding the Pi lock past Pi process exit and producing spurious 409s on the next request.
- **Pedagogy block in `CLAUDE.local.md` and `memory/study-context.md`** also lost the legacy "markdown plan or JSON plan" dual references in favor of "JSON via `claw-cli` only" — the markdown plans were retired in Claw's group memory on 2026-05-14.

### Fixed
- **AGENTS.md was always falling back to the placeholder.** `agent/sandbox.go` skipped `claw-cli memory load` whenever `Config.ClawCLIPath == ""`, and `CLAW_CLI_PATH` was unset in `.env`. Setting `CLAW_CLI_PATH=/usr/local/bin/claw-cli` in `.env` plus the `/usr/local/bin/claw-cli` symlink restored the load — Pi now sees real course context (user profile, observations, course memory, active feedback rules) instead of the "No memory loaded for this session" stub.
- **Pi was reading the wrong plan.** Two plan stores had silently diverged: `data/plans/<course>.json` (UI-canonical) and `~/stack/nanoclaw-v2/groups/dm-with-hiroji/memory/courses/<course>/study-plan.md` (Pi's habit). 2026-05-14: the markdown plans were renamed `.retired-2026-05-14.md`; AGENTS.md template + per-turn `<plan_state>` prepend force Pi to use `claw-cli plan status`/`plan toggle` only.

### Resolved (no code change)
- **Session topic auto-rename.** Verified `handler/chat_v2.go:autoTopic` already truncates to ≤60 runes at a word boundary and appends "…". Decision: keep status quo — server-side truncation is the guaranteed fallback, Pi's `claw-cli session topic` override stays as the quality upside. Item closed from ROADMAP "Next".

### Ops
- `AGENT_MODEL` removed from `.env` so Pi falls back to `LLM_MODEL=glm-5.1`. The previous `AGENT_MODEL=kimi-k2.6` reliably returned `content:[]` (empty response, zero usage tokens) on its second-of-two API calls per turn — confirmed via direct gateway test that kimi-k2.6 works for single calls but breaks on tool-result continuations. glm-5.1 handles tool-use protocol correctly.
- `/usr/local/bin/claw-cli` symlink added (owned by root) so the bare `claw-cli` command works in Pi's bash environment without depending on PATH inheritance.

## [2026-05-11]

### Added
- **Pi agent runtime.** Each chat session now runs through `@earendil-works/pi-coding-agent` (Pi), a subprocess-based agent with a bash tool surface. `/chat-v2` endpoint replaces `/chat` for sessions in Pi mode. Pi is spawned per-turn via `agent.RunPi`; SSE events (`token`, `reasoning`, `tool_start`, `tool_end`, `skill_start`, `compaction`, `model_change`, `done`, `error`) stream directly to the browser.
- **Per-session Pi sandbox.** `SandboxManager` creates `data/agent-sessions/<id>/` with a generated `AGENTS.md` (via `claw-cli memory load --session --course`), a `notes/` scratch dir, and a symlink to `data/agent-out/`. Sandboxes are swept after 7 days of inactivity.
- **`claw-cli session topic` subcommand.** Sets a descriptive topic on any session by ID. Accepts `--session-id` and `--topic` flags; reads/writes study.db via `App.UpdateSessionTopic`.
- **Session topic auto-rename on first turn.** `handleChatV2` detects the first message in any "General" session and immediately derives a title by truncating the user message (≤60 runes, word-boundary). Sends a `session_topic` SSE event so the sidebar and header pill update without a page reload. Pi can still override the topic during its run.
- **Inline reasoning stream.** Thinking tokens now render inline between the answer segments that surround them rather than in a single pre-allocated block at the top. Each reasoning span is a `<details class="thinking-inline">` collapsed by default; answer tokens go into `<div class="answer-segment">` divs. Frontend tracks `currentSegmentType` across both event types.
- **Tool calls visible in chat UI.** `tool_start` and `tool_end` SSE events render as collapsible tool panels inside the answer timeline, showing the tool name, input summary, output summary, and success/failure indicator.
- **Session topic inline rename.** Pencil icon next to each session name in the sidebar; click converts the label to an `<input>`, `Enter` or blur commits via `PATCH /api/sessions?id=`. Topic is updated in-memory and the header pill refreshes.
- **Pomodoro timer in header.** `#pomodoro-btn` in the header actions area shows a 25:00 countdown. Click starts/pauses, double-click resets. Switches to 5-min break after each focus block. `font-variant-numeric: tabular-nums` prevents width jitter.

### Changed

- File tools (`read_file`, `list_files`) now accept vault-relative paths (joined to `VaultRoot`) in addition to absolute paths. Tool descriptions updated to document relative-path support.
